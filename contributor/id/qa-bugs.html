Berurusan dengan bug

Menggunakan tag untuk laporan bug Xubuntu

Agar kami dapat dengan mudah menemukan bug untuk Xubuntu selama siklus pengembangan, ada beberapa tag yang dapat Anda tambahkan ke laporan bug Anda. Kami menganggap bahwa tag untuk nama kode siklus disertakan, jika tidak, tambahkan saja.

  • ppa: tambahkan tag ini ketika Anda melaporkan bug terhadap paket dari salah satu dari 3 PPA pengembangan

  • kegunaan: tambahkan tag ini jika masalah yang Anda laporkan bukan bug standar, tetapi salah satu yang mungkin dianggap sebagai daftar harapan, seperti beberapa pilihan hasil di jendela hasil Catfish

Pelaporan bug normal

Siapa pun yang menggunakan Xubuntu dapat mengajukan (atau dengan kata lain, melaporkan) bug, bahkan tanpa pengalaman sama sekali. Mengarsipkan bug penting karena dalam beberapa kasus, bug dapat diketahui tanpa disadari oleh pengguna akhir.

Before submitting a bug, you should look at the existing bug reports and release notes to verify the bug hasn’t been reported already. If the bug has not already been reported, you should file a new bug report. It is sensible to read through the bug reporting guidelines before filing your first bug.

In the most situations, it is easiest to file a bug by opening a new Terminal and type ubuntu-bug package-name, where package-name is the package you want to file bug against. If you don’t know which package you should file the bug against, refer to the instructions on finding the right package. When filing the bug, it is better to have too much information than too little.

If you are unsure of the package name to report against, you can use your mouse to click on the application window; in a terminal run ubuntu-bug -w, click to close the message window and then click on the application window.

Melaporkan bug dengan PPA

When you encounter a bug with a package from a PPA, you’ll need to file the bug report manually. Once you’ve got the offending package name, go to https://bugs.launchpad.net/ubuntu/+source/PACKAGENAME/+filebug.

Crash logs related to the package can be found in /var/crash/. This directory sometimes requires superuser (sudo) permissions to view and extract the contents. These logs can be particularly valuable to include in your bug report, but be sure to review them as they may contain sensitive or personally identifiable material.

Pastikan untuk mengikuti pedoman umum di atas, dan juga menambahkan tag ppa sehingga jelas bagi pengembang bahwa ini adalah paket yang tidak didukung.

Triaging bugs

Triaging bugs is getting bug reports in a state where they are useful for developers by making sure bug reports have useful titles, descriptions, appropriate logs and more. To get started, read the page on triaging bugs in the Ubuntu wiki.

After you have made yourself familiar with the aspects of triaging bugs, you can start triaging Xubuntu bugs. Please note that the bug status and importance can only be changed by members of the Ubuntu Bug Control team #8211 in the beginning, you will need to report triaged bugs for the team in #ubuntu-bugs on Libera. After you have demonstrated your ability to triage bugs, you will gain more responsibilities from the Bug Control team.

Ketika Anda mendapatkan pengalaman dalam triaging bug Xubuntu, Anda mungkin ingin melihat bug baru yang menyebutkan Xubuntu. Triaging bug baru direkomendasikan bagi mereka yang lebih terbiasa dengan triaging dan Xubuntu secara umum, karena tidak semua bug baru yang menyebutkan Xubuntu sebenarnya adalah bug Xubuntu.

Meneruskan bug di bagian hulu

Ketika bug diteruskan ke hulu, semua pengguna perangkat lunak yang relevan akan mendapat manfaat dari pekerjaan debugging, triaging dan patching yang dilakukan oleh pemicu bug dan pengembang.

Once it has been determined that a bug is not caused by a change in Xubuntu, bugs can be forwarded to their respective upstream projects to be reviewed. For example, all appropriate Xfce bugs should be filed in the Xfce bug tracker. In addition, to track the status of the bug in Ubuntu, the bug in Launchpad should be linked with the upstream bug.

The Ubuntu wiki has an extensive page on forwarding bugs upstream along with instructions on how to report bugs in the upstream bug trackers. Detailed information on the way to add an upstream bug to Launchpad can be found on the Bugs/Watches page on the Ubuntu wiki.
