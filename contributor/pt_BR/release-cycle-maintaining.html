Mantendo

Após um lançamento ser feito,é certo que bugs e problemas serão relatados. O período entre o lançamento e o início do próximo ciclo de desenvolvimento é o momento ideal para performar Stable Release Updates (SRU) para bugs significativos contanto que os desenvolvedores não esqueçam de consertar o problema na próxima versão também, uma vez que o repositório abra. quando o próximo ciclo de lançamento é iniciado, o foco primário será na próxima versão e a gravidade/importância dos SRU candidatos será mais importante ao decidir se um SRU será efeituado ou não.
