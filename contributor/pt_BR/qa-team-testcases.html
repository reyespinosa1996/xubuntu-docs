Trabalhando com casos de teste

Informação sobre o método básico de trabalhar com casos de testes pode ser encontrada no manual de casos de testes na página da Equipe de controle de qualidade do Ubuntu.

Nósestamos apenas preocupados com um conjunto específico de tarefas: puxando a branch,fazendo edições e então enviando as mudanças que precisamos para a branch principal. Nós temos pessoas na equipe de Administradores de Casos de Teste para os casos de testes LP, além disso qualquer membro da nossa equipe de lançamento pode editar o rastreador, isso ajuda a garantir que alterações sejam movidas aos rastreadores rapidamente.

Para editar um caso de teste:

  • Relatara alteração requerida como um bug contra o projeto de caso de teste

  • Atribua-se ao bug

  • Criar uma branch local: bzr branch lp:ubuntu-manual-tests

  • Faça alterações no caso de teste em sua branch local

  • Enviar as alterações: bzr commit -m "Fix LP bug #BUGNO."

  • Enviar a uma branch remota pessoal: bzr push lp:~username/ubuntu-manual-tests/bug-BUGNO

Uma vez enviada a uma branch remota pessoal, propor uma alteração para mesclagem - Enviando propostas de mesclagem.

Responda á quaisquer solicitações para alterações quando pedidas pelos administradores de casos de testes para conseguir a alteração requerida em tempo hábil.
