Enviando a repositórios Git

Quando tiver trabalhado em suas alterações e quiser movê-las a um repositório remoto, faça o seguinte na branch local:

  • Para renomear ou mover arquivos, executegit mv ANTIGO NOVO. Isso ajudará manter o diff resultante significativo,enquanto Git deverá descobrir isto automaticamente.

  • Para remover arquivos, executegit rm ARQUIVO. Isso também impede o rastreio de arquivos especificados,embora seja redundante se todas as alterações estão preparadas para depois.

  • Se tiver adicionado arquivos, executegit add .no diretório root da branch. Isto iniciará o rastreio de quaisquer arquivos ainda não rastreados, e também preparará quaisquer alterações.

  • Para verificar o estado da branch, execute git status.

  • Faça o commit das suas alterações ao executar git commit -a. Isto irá preparar qualquer alteração,e irá abrir um editor de texto para inserir a mensagem de confirmação.

  • Após ter feito um ou mais commits, executegit push LOCALIZAÇÃO BRANCH para mover suas alterações a um repositório remoto.

A mensagem de commit deverá resumir todas as alterações que você fez após o último commit. Você pode especificá-la diretamente usando -m MENSAGEM.
