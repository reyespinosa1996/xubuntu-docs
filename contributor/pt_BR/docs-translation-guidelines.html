Diretrizes de tradução para pacotes do Xubuntu

Estas diretrizes se aplicam principalmente a documentação do Xubuntu. Elas geralmente podem ser aplicadas a qualquer tradução com leve modificações. Se não tiver certeza,peça assistência aos membros da equipe de Documentação.

O Equipes de tradução de linguagens específicasgeralmente quer encorajar alguns acordos; por favor contate a equipe de tradução de linguagem específica apropriada antes de traduzir.

Mais diretrizes gerais de tradução podem ser encontradas em Guia de traduções do Launchpad.

Manter a validade da tradução

Tags não devem ser traduzidasQuando ver uma tag na string fonte (por exemplo: <xref linkend="&xubuntu-web;"/>"), a tag e qualquer coisa dentro da própria tag não deve ser traduzido. É importante copiar e colar essas tags exatamente como aparecem na string fonte. Observação: a exceção a isso é quando uma tag contém uma URL que possui uma versão equivalente,traduzida. Nesses casos a tradução da URL ficará a critério do tradutor.

Entidades não devem ser traduzidas.Quando ver uma frase expressada como "&gt;"  ou "&xubuntu-web;", não a altere.

Sempre mantenha a ordem das tags.Quando ver tags diferentes agrupadas umas nas outras(por exemplo: "<menuchoice><guimenu>System</guimenu><guimenuitem>Administration</guimenuitem><guimenuitem>Users and Groups</guimenuitem></menuchoice>"), você sempre deverá preservar a ordem cuidadosamente - copiando e colando da linguagem original(Inglês) é a melhor maneira de garantir que você faça isso corretamente.

Linguagens de programação diferentes e software podem ter sintaxes variáveis diferentes: tenha certeza que está familiar com a sintaxe variável apropriada do software ao traduzir. Mais informações podem ser encontradas na string de tradução do comentário.

Siga acordos bons e existentes

Use os nomes de aplicativo e rótulos encontrado na interface (gráfica).Ao traduzir nomes de aplicativos e rótulos de interface gráficas,use o termo ou nome usado na interface gráfica. Isso facilitará o usuário a seguir as instruções. Executar o sistema tanto na linguagem de tradução predestinada quanto em inglês pode ajudar nisso.

Não altere a quantidade de espaçamento.Ao traduzir, não altere a quantidade de espaçamento dentro ou fora das tags. Manter o espaçamento como está torna a revisão técnica da tradução mais fácil.

Não altere o tipo de aspas usadas dentro das tags.As aspas duplas ( " ) não possuem a mesma significância em um DocBook, XHTML ou tag XLM como as aspas ( « » ). Usar tipos diferentes de aspas pode levar a problemas de validade também.

Nunca altere uma tag para outra.Se achar que uma tag na fonte é inválida ou não semântica, registre um bug contra o próprio pacote invés de alterá-la na tradução.
