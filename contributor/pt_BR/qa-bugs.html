Lidando com bugs

Usando tags para reportar bugs do Xubuntu

Para que possamos facilmente encontrar bugs do Xubuntu durante um ciclo de desenvolvimento, há algumas tags que você pode adicionar aos seus relatórios de bug. Nós assumimos que uma tag para o codinome do ciclo está incluída,caso não esteja por favor adicione uma.

  • ppa: adicione esta tag quando tiver relatado um bug contra um pacote de um dos 3 PPAs de desenvolvimento

  • usabilidade: adicione esta tag se o problema que você está relatando não é um bug comum, but um que pode ser considerado na lista de desejos,como seleções múltiplas de resultados da janela de resultados Catfish

Reportando bug normal

Qualquer um usando Xubuntu pode registrar (ou em outras palavras,relatar) bugs, mesmo tendo nenhuma experiência. Reportar bugs é importante pois em alguns casos, bugs podem passar despercebidos a menos que um usuário os registre.

Antes de enviar um bug, você deverá olhar em reportes existentes de bugse nas notas de lançamento para verificar se o bug já não foi relatado. Se o bug já não foi relatado,você deve registrar um novo reporte de bug. É sensato ler diretrizes de reporte de bugsantes de registrar o seu primeiro bug.

Na maioria das situações,é mais fácil registrar um bug abrindo um terminal e digitando ubuntu-bug nome-pacote,onde nome-pacote é o pacote que você registrar um bug contra. Se não souber qual pacote você deve registrar um bug contra, refira-se ás instruções em Encontrando o pacote certo. Ao registrar o bug,é melhor ter informação demais do que pouca informação.

Se não tiver certeza do nome de pacote a relatar, você pode usar o seu mouse para clicar na janela de aplicação; em um terminal execute ubuntu-bug -w, clique para fechar a mensagem de janela e então clique na janela da aplicação.

Reportando bugs com PPAs

Quando encontrar um bug em um pacote de PPA. você precisará registrar o relato de bug manualmente. Quando tiver o nome do pacote faltoso, vá para https://bugs.launchpad.net/ubuntu/+source/PACKAGENAME/+filebug.

Registros de falhas do pacote podem ser encontrados em /var/crash/. Este diretório as vezes requer permissões administrador(sudo) para ver e extrair os conteúdos. Esses registros podem ser particularmente valiosos no seu relato de bug, mas tenha certeza de revisá-los pois podem conter material sensível ou pessoalmente identificável.

Tenha certeza de seguir as diretrizes gerais acima,e também adicione a tag ppa para tornar claro aos desenvolvedores que esse é um pacote não suportado.

Triagem de bugs

Triagem de bugs é receber relatórios de bugs em um estado quando são úteis aos desenvolvedores por garantir que relatórious de bugs tenham títulos úteis, descrições, registros apropriados e mais. Para começar,leia a página triagem de bugsda Ubuntu wiki.

Após familiarizar-se com os elementos da classificação de bugs, poderá iniciar a classificação Bugs do Xubuntu. Favor observar que o status do bug e importância podem ser alterados apenas pelos membros da Controle de Bugs do Xubuntuequipe#8211 no início,você precisará relatar bugs classificados para a equipe em #ubuntu-bugs no Libera. Após demonstrar a sua habilidade de classificar bugs,você irá ganhar mais responsabilidades da equipe de Controle de Bugs.

Conforme você ganha experiência ao classificar bugs do Xubuntu, você pode querer dar uma olhada em novos bugs que mencionam o Xubuntu. Classificar novos bugs é recomendado aqueles que estão mais familiarizados com a classificação e o Xubuntu geralmente, já que nem todos os novos bugs mencionando Xubuntu são verdadeiramente bugs do Xubuntu.

Encaminhando bugs ao upstream

Quando bugs são encaminhados ao upstream, todos os usuários do software relevante se beneficiarão do trabalho de depuração,classificação,e correção feito pelos classificadores de bugs e desenvolvedores.

Ao ser determinado que um bug não é causado por uma alteração no Xubuntu, bugs podem ser encaminhados ao upstream de seus respectivos projetos para serem analisados. Por exemplo, todos os bugs apropriados do Xfce devem ser registrados no rastreador de bugs do Xfce. Além disso, para rastrear o status de um bug no Ubuntu, o bug no Launchpad deve ser vinculado ao bug no upstream.

A Ubuntu Wiki tem uma página extensiva em encaminhando bugs ao upstream juntamente com instruções sobre como relatar bugs nos rastreadores de bug upstream. Informação detalhada na forma de como adicionar um bug do upstream no Launchpad pode ser encontrado na página Bugs/Vigias da Ubuntu wiki.
