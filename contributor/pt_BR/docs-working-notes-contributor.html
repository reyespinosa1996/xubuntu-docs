Observações na documentação do colaborador

Por vários motivos,a documentação do colaborador possui uma estrutura mais complexa que a documentação do usuário. Particularmente,preste atenção ao seguinte quando estiver trabalhando com a sub-equipe de documentação:

  • Se quiser criar um parágrafo introdutório que seja mostrado na página principal da sub-equipe, simplesmente não o envolva dentro de um <section>elemento. De outra forma ele será fragmentado em um arquivo HTML separado.

  • Se quiser que uma subseção seja visível no índice,adicione o atributo <role="toc">para o elemento <section>. Para um exemplo real, veja os arquivos qa-*.

  • As seções são fragmentadas até o primeiro nível; Isso significa que todos os ascendentes diretos<section> dos elementos <chapter> serão fragmentados em arquivos HTML separados. Se quiser evitar isso (Se seu capítulo está muito curto por exemplo). Insira o <?dbhtml stop-chunking?>após o <title>elemento do seu capítulo. Observe que fazer isso não irá impedir o Índice de ser estampado em cada seção. Você pode usar o <simplesect>elemento novamente para ocultar seções do Índice. Se fizer isso,tenha certeza que você não insira quaisquer elementos após os elementos <simplesect>em seu capítulo, já que isso invalidará o documento.
