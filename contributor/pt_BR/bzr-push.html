Enviando por Push para repositórios Bzr

Quando tiver trabalhado em suas alterações e quiser movê-las a uma branch remota, faça o seguinte na branch local:

  • Para renomear ou mover arquivos, executebzr mv ANTIGO NOVO. Isso ajudará manter o diff resultante significativo, e também funcionará após o fato.

  • Se tiver adicionado arquivos, executebzr add .na branch root. Isto iniciará o rastreio de quaisquer arquivos na filial que estão indeterminados no momento.

  • Se tiver removido arquivos, executebzr remove.Na maioria dos casos,isso deverá automaticamente parar o rastreio de arquivos que não estão mais na branch.

  • Para verificar o estado da branch, execute bzr status.

  • Faça o commit das suas alterações ao executarbzr commit.Isto irá abrir um editor de texto para inserir a mensagem de confirmação.

  • Após ter feito um ou dois commits, executebzr push LOCALIZAÇÃOpara mover suas alterações a uma branch remota.

A mensagem de commit deverá resumir todas as alterações que você fez após o último commit.Você pode especificá-la usando-m MENSAGEM,e vincular um reporte de bug ao commit usando --fixes lp:NÚMERODOBUG.

Quando estiver trabalhando em uma branch e fazendo várias alterações que não são relacionadas umas as outras,é recomendado fazer vários commits. Isso ajuda manter o log de commits limpo e torna mais fácil a procura de qualquer problema que possa ter sido introduzido.

Quando mover uma alteração para uma branch remota pela primeira vez, terá que especificar a localização da branch, Na maioria dos casos,ela irá seguir o formato lp:~username/projectname/branchname. Isso moverá o seu código para uma branch remota pessoal, Ao mover alterações repetidamente, você poderá omitir a localização da branch, já que Bzr terá salvo a localização mais recente na configuração da branch local.
